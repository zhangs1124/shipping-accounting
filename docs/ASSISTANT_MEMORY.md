2. **開發前確認協議**：
   - 階段 1：研究並提出 `implementation_plan.md`。
   - 階段 2：User 認可計畫內容（LGTM）。
   - 階段 3：**詢問 User 是否可以開始「施作/修改」**。
   - 階段 4：獲得明確授權後，才執行 `write_to_file` 或 `replace_file_content`。
3. **回覆標註**：每則回覆開頭均須標註 `[已確認：不自動改程式，僅供範例並註明行號]` 及模型名稱。
4. **溝通語言**：全程使用繁體中文。
