# 開發經驗：使用 MCP Playwright 測試正式環境伺服器

## 背景
在正式環境中，系統運行於 HTTPS (Port 8443) 並使用自簽名 SSL 憑證。這導致一般自動化測試工具（如 Playwright）在導航時會因為安全性憑證無效而中斷。

## 解決方案
使用 MCP Playwright 進行手動導航與繞過安全警告：

1. **初始導航**：使用 `mcp_playwright_browser_navigate` 指向 `https://127.0.0.1:8443`。
2. **處理 SSL 警告**：
   - 識別頁面上的「進階」按鈕並點擊。
   - 識別「繼續前往 127.0.0.1 網站 (不安全)」連結並點擊。
3. **驗證**：進入登入頁面後，擷取螢幕截圖以確認 UI 渲染正常。

## 經驗心得
- 對於自簽名憑證，如果 MCP 工具未提供 `ignoreHTTPSErrors` 選項，可以透過 Playwright 模擬點擊瀏覽器的「進階」與「繼續」按鈕來達成目的。
- 使用 `mcp playwright` 相比內建瀏覽器，更適合在 headless 環境或遠端開發環境中進行視覺化驗證。
