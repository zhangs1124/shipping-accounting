# 開發經驗：使用 MCP Playwright 測試正式環境伺服器

## 背景
在正式環境中，系統運行於 HTTPS (Port 8443) 並使用自簽名 SSL 憑證。這導致一般自動化測試工具（如 Playwright）在導航時會因為安全性憑證無效而中斷。

## 解決方案
1. **正式環境 (SSL)**：使用 `run_prod.bat`，需處理 `ERR_CERT_AUTHORITY_INVALID`。
2. **切換至 Port 8000**：若需更便捷的測試環境，可關閉正式環境程序並改用 `run_app.bat`。

## 執行切換紀錄 (2026-04-21)
- **原因**：為了簡化測試流程並符合原先約定的 Port 8000 運行規範。
- **動作**：
  1. 使用 `taskkill` 強制停止 Port 8443 與 8000 的 Python 程序。
  2. 重新啟動 `run_app.bat`。
  3. 驗證 `http://127.0.0.1:8000` 存取正常。

## 經驗心得
- 在開發與頻繁測試階段，使用 `run_app.bat` (Port 8000) 配合熱重載功能，比處理正式環境的 SSL 跳轉與憑證警告效率更高。
- 透過 `mcp playwright` 測試時，使用 `127.0.0.1` 往往比 `localhost` 更穩定。

## 測試環境
- **URL**: https://127.0.0.1:8443
- **測試工具**: MCP Playwright
- **測試時間**: 2026-04-21
