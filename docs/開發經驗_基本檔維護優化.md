# 開發經驗：基本檔維護功能優化 (2026-03-25)

## 任務背景
使用者反饋系統中「基本檔維護」做得不夠理想。經過分析，發現各功能頁面實作風格不一：部分使用現代化的 AJAX Modal（如船舶與收費項目），部分仍維持舊式的全頁跳轉（如客戶管理），這導致了操作體驗的斷裂。此外，進出港項目的 AJAX 介面存在欄位遺漏的隱漏。

## 優化重點

### 1. 客戶管理介面重構
- **從跳轉到彈窗**：移除 `customers/new.html` 與 `edit.html`，統一在 `list.html` 內透過 Modal 進行維護。
- **後端 API 完善**：補全 `/customers/api` 下的 Create, Update, Delete JSON 路由。
- **新增搜尋過濾**：在標題欄增加搜尋輸入框，利用 JavaScript 實現本地端的表格過濾，提升資料量大時的檢索效率。

### 2. 進出港項目 Bug 修復
- **欄位修正**：修復了 `task_categories` 在新增/編輯時漏掉 `task_group` (任務分組) 的邏輯問題。
- **無縫更新**：將操作成功後的處理從 `location.reload()` 改為動態更新表格 Row (DOM 操作)，大幅提升流暢感。

### 3. 事件委派與安全傳參
- **Data Attributes 模式**：捨棄傳統在 `onclick="..."` 中傳遞大量字串參數的做法（易導致 JS 語法報錯），改為使用 HTML5 的 `data-*` 屬性儲存資料，並透過 `dataset` 在 JS 中統一取得。
- **事件監聽器**：改用動態事件委派 `addEventListener('click', ...)`，確保 AJAX 新增的表格行也能正確響應點擊事件。

## 技術收穫
1. **介面一致性的重要性**：在商用系統中，統一的操作模式（如全部使用 Modal）比華麗但零散的技術實作更為重要。
2. **AJAX 原生實作**：不依賴大型框架，純使用 Vanilla JS + Fetch 也能建構出高效且維護性強的 CRUD 介面。
3. **錯誤處理優化**：透過 API 返回 4xx 狀態碼與 JSON Error Message，並在 Modal 內即時顯示，能顯著改善用戶體驗。

## 待優化點
- 未來可考慮將各個基本檔維護頁面的 JS 邏輯進一步模組化，萃取出 `CrudTableHelper` 類別以減少重複程式碼。
