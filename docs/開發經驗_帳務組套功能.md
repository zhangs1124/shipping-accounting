# 開發經驗：帳務組套 (Charge Packages) 功能實作

## 1. 功能需求
為了加速帳務登錄速度，系統需要支援預設的費用清單（組套）。例如針對「船東」或「代理」身份，一鍵自動帶入多筆預設費用項目。

## 2. 實作重點

### 2.1 資料庫設計 (Schema)
採用多對多關聯結構：
- `charge_packages`: 存儲組套名稱與描述。
- `charge_package_items`: 關聯組套與收費項目，並支援預設數量設定。

### 2.2 後端架構
- **Router 分離**: 新增 `routers/charge_packages.py` 專門處理組套的 CRUD 維護。
- **批次處理邏輯**: 在 `routers/invoice_lines.py` 實作 `apply-package` 接口，遍歷組套項目並轉換為 `InvoiceLine` 實體，同步更新帳單總金額。

### 2.3 前端整合
- **設定頁面**: 採用與系統一致的卡片式設計，支援動態從組套中加入/移除費用項目。
- **套用介面**: 在帳單明細頁面頂部加入明顯的「📦 套用費用組套 (批次帶入)」區塊，並加上 JS 確認視窗防止誤點。

## 3. 遇到的問題與解決方案 (Bug Fix)
- **問題**: 實作完成後訪問 `/charge-packages` 出現 `404 Not Found`。
- **分析**: 發現 Windows 環境下有多個 Python 殘留程序 (Residual Processes) 佔用了 Port 8000，導致舊的程式碼實體仍在記憶體中運行。
- **解決**: 使用 `taskkill /F /IM python.exe` 徹底清除後重新執行 `run_dev.bat` 恢復正常。

## 4. 檔案變更清單
- `models.py`: 新增組套相關模型。
- `main.py`: 註冊新路由。
- `routers/charge_packages.py`: 組套管理邏輯。
- `routers/invoice_lines.py`: 實作批次套用邏輯。
- `templates/charge_packages/`: 新增列表與編輯模板。
- `templates/invoices/detail.html`: 整合套用介面。
- `templates/base.html`: 加入導覽選單。
