# 開發經驗紀錄 - 檢疫項目基本檔維護功能

## 1. 需求背景
「進出港」作業涉及預報、現場、檢疫、財務等多個環節。為了讓使用者能動態調整所有任務的名稱、分組、規費金額及顯示順序，實作此維護介面。

## 2. 技術設計
### 2.1 資料模型應用
- 統一管理 `TaskCategory` 模型，支援按分組排序。

### 2.2 前端互動 (Modal CRUD)
- **多功能彈窗**: 支援編輯分組標籤及規費。

## 3. 實作重點
- **刪除防呆**: 聯動檢查 `VoyageTaskLog`。如果某個檢疫項目已經有航次使用過了（已記錄時間），則禁止物理刪除，改為提示使用者將其設為「停用」。
- **排序與狀態**: 提供 `display_order` 與 `is_active` 欄位，確保介面呈現的靈活性。

## 4. 後續規劃
- 若主管未來需要維護其他組別（如「財務」或「現場」）的項目，可基於現有的 `task_categories` 路由輕易擴充。
