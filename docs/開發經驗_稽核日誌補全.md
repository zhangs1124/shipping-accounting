# 開發經驗：進出港與提醒中心稽核日誌補全

## 問題背景
用戶發現進出港任務的「壓紀錄」、「清空」以及「加入提醒」等動作，在系統審核日誌 (Audit Log) 中完全沒有紀錄。這對於需要追蹤操作人員行為的帳務系統來說是一個嚴重的缺口。

## 技術分析
1. **自動監聽機制**：系統使用 SQLAlchemy 的 `before_flush` 監聽器來自動紀錄資料表異動。
2. **人為排除**：在 `utils/audit_logger.py` 中，為了效能或避免噪音，開發初期將 `voyage_task_logs` 與 `reminders` 設入了 `AUDIT_EXCLUDE_TABLES`。
3. **批量操作限制**：SQLAlchemy 的 Session 監聽器無法捕捉到 `query.delete()` 這種直接產生 SQL 的批量動作。

## 解決方案
1. **解封資料表**：將 `voyage_task_logs` 與 `reminders` 從排除清單移除，立即獲得了 90% 的日誌覆蓋率（包含所有單筆修改與新增）。
2. **手動補強**：在 `purge_voyage_tasks` 這類特殊 API 中，先呼叫 `log_action` 紀錄意圖，再執行 SQL 動作。
3. **上下文確保**：確認所有 API 皆經過驗證權限層，確保 `current_user_id` 與 `ip_address` 能被正確存入 ContextVar。

## 成果
現在管理員可以清楚地在「操作紀錄」報表中，看到誰在何時修改了哪一筆進出港任務的時間，或是在何時為哪筆任務設定了提醒。這大大提升了系統的可追蹤性。
