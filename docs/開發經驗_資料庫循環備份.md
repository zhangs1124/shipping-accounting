# 船務部帳務系統：開發經驗 - 資料庫循環備份實作 (SQLite)

## 1. 背景與需求
為了因應 SQLite 在 Windows 環境下的資料安全性，我們需要一個自動化的備份方案：
- **頻率**：每小時執行一次。
- **保留期限**：最近 7 天。
- **技術考量**：必須支援 WAL 模式下的線上備份，不可鎖定執行中的資料庫。

## 2. 實作重點
### 2.1 線上備份工具
採用 Python `sqlite3` 模組內建的 `backup()` API：
- 此方式能優雅地將來源資料庫（包含 WAL 內容）複製到新的備份檔。
- 優於直接文件複製，可避免 `database is locked` 或損毀風險。

### 2.2 自動化排程 (APScheduler)
整併至 `main.py` 的背景任務系統：
- `cron (minute=0)`：定時每小時整點觸發。
- `date` (startup)：系統啟動時首做一次，確保環境（目錄、權限）一切正常。

### 2.3 檔案輪轉 (Rotation)
檔案命名採 `shipping_backup_YYYYMMDD_HHMM.db` 格式。
清理邏輯：每次備份成功後，掃描備份目錄並刪除超過 `168` 小時 (7天) 的舊檔案。

## 3. 經驗心得與規範 (Memory Update)
- **備份檔存放**：獨立存放於 `backups/` 目錄。
- **作業規範**：重要功能的開發與「施作」必須先經過 Plan 階段，並獲得 User **明確授權「直接修改」** 後方可執行。
- **溝通記錄**：此次開發確認了「不自動修改程式」的核心溝通規範，並記錄於 [ASSISTANT_MEMORY.md](ASSISTANT_MEMORY.md)。

## 4. 驗證結果
已通過 `test_backup_logic.py` 手動測試與系統啟動時的自動備份測試，成功產生備份並能正確清理過期檔案。
